## BUG 1 — ESO Pods ImagePullBackOff (ECR image không có)

**Triệu chứng:** `external-secrets`, `external-secrets-cert-controller`, `external-secrets-webhook` ở trạng thái `ImagePullBackOff`.

**Root cause:** `operators.tf` chỉ set `image.repository` cho main controller. `certController.image` và `webhook.image` mặc định về `ghcr.io/external-secrets/external-secrets` — không accessible trong VPC NAT-less.

**Fix (Terraform):** Thêm vào `infra/environments/sandbox/services/operators.tf`:
```hcl
certController = {
  image = {
    repository = "${local.ecr_registry}/external-secrets/external-secrets"
    tag        = "v0.9.13"
  }
}
webhook = {
  image = {
    repository = "${local.ecr_registry}/external-secrets/external-secrets"
    tag        = "v0.9.13"
  }
}
```

## BUG 2 — ESO ClusterSecretStore ValidationFailed (STS global endpoint)

**Triệu chứng:** `ClusterSecretStore aws-secrets-manager` ở trạng thái `ValidationFailed` với lỗi `sts.amazonaws.com: i/o timeout`.

**Root cause:** ESO dùng STS **global** endpoint `sts.amazonaws.com` theo mặc định. VPC endpoint chỉ route **regional** endpoint `sts.us-east-1.amazonaws.com`. Dù VPC endpoint tồn tại (`vpce-05399e0bf8d89256d`), global endpoint traffic không được route qua nó.

**Fix:** Thêm env var vào ESO deployment trong `operators.tf`:
```hcl
env = [
  {
    name  = "AWS_STS_REGIONAL_ENDPOINTS"
    value = "regional"
  }
